منصة SaaS متكاملة لإدارة مراكز التدريب

منصة إدارية احترافية تغطي دورة حياة المتدرب بالكامل — من التسجيل في رابط مُتحكَّم، مروراً بالقبول، ثم التعيين في مجموعات، ومتابعة الحضور ببصمة إلكترونية، والإجازات المرضية برقم رسمي، والمخالفات المنظمة بخمس درجات، والرواتب المحسوبة تلقائياً، حتى التجديد أو إنهاء العقد. مصمَّمة لفريقين من مطوري Full-Stack، قابلة للتوسع لآلاف المتدربين بدون تراجع في الأداء.

مشروع مركز التدريب الرقمي — نظرة شاملة

هذه الوثيقة تشرح البناء الكامل لنظام إدارة معهد ريادة الغد — موجَّهة للعميل، لفريق التطوير، ولقرارات الإدارة العليا. كل مرحلة موثَّقة بصفحة مستقلة ومترابطة مع باقي المراحل.

🎯
طبيعة النظام
منصة إدارية تشغيلية (Operational Admin SaaS)

نظام ويب متعدد المستأجرين (Multi-Tenant) يخدم معهداً واحداً رئيسياً له عدة شركات شريكة. كل شركة لها أقسام، وكل قسم به برامج تدريب، وكل برنامج يُقسَّم إلى مجموعات. يُدار النظام مركزياً من المعهد، وتملك كل شركة رؤية محدودة لبيانات متدربيها فقط.

🧭
أهداف الأعمال
لماذا النظام موجود أصلاً؟
  • ✓ تقليل العمل اليدوي في الحضور والإشعارات والتقارير بنسبة 80%
  • ✓ منع الأخطاء البشرية في الرواتب والمخالفات
  • ✓ إثبات الالتزام تجاه الشركات الشريكة بتقارير موثَّقة
  • ✓ رفع جودة تجربة المتدرب عبر بوابة خاصة
  • ✓ إدارة دقيقة للعقود وتجديدها أو إنهائها

الرؤية

لماذا "ريادة الغد" نظام مختلف؟

لا يوجد في السوق نظام عربي متخصص يجمع: (1) بصمة + حضور إلكتروني، (2) عقود شركات، (3) مخالفات بخمس درجات، (4) رواتب فردية، (5) منصة متميزين، في منظومة واحدة.

🎓 متدرب في المركز

بوابة شخصية تعرض: حالة التدريب، سجل الحضور الشهري، الراتب المحدَّث، الإجازات، المخالفات الموقَّعة إلكترونياً، طلب تعديل البيانات، تجديد الهوية، شارة التميز.

👨‍💼 موظف إداري

لوحة تحكم بصلاحيات دقيقة لكل صفحة وكل إجراء. قد يُقيَّد بنطاق الجنس (ذكور فقط / إناث فقط) ونطاق الشركات، مع تسجيل كل إجراء في Audit Log.

🏢 شركة شريكة

تستقبل تقارير حضور شهرية موثَّقة، إشعارات قبل انتهاء العقد بـ 60 و 30 يوماً، تقارير المخالفات، وقائمة المتميزين من متدربيها.

💡
الرؤية طويلة المدى: أن يكون النظام قابلاً للاستنساخ (White-Label) لمعاهد تدريب أخرى، بنفس البنية، مع تخصيص الألوان والشعار فقط. قاعدة البيانات مصمَّمة من اليوم الأول لدعم ذلك.

🔐

نطاق المشروع

ما يشمله المشروع — وما لا يشمله

تحديد النطاق بوضوح يمنع التمدد غير المدروس (Scope Creep) ويوضِّح للعميل ما سيُسلَّم فعلياً.

✅ ما يشمله المشروع

  • 🌐 واجهة إدارة كاملة — 19 صفحة بصلاحيات دقيقة
  • 🎓 بوابة متدرب — 10 صفحات + صفحة التسجيل العام
  • 🧑‍🏫 بوابة مدرب — عرض مجموعاته + تسجيل حضور + مخالفات
  • 🔗 ربط جهاز البصمة عبر API/SDK مع مطابقة تلقائية
  • 📱 إشعارات واتساب + إيميل (SMTP) بقوالب ديناميكية
  • 📊 Excel Import/Export لكل الكيانات الرئيسية
  • 📄 تصدير PDF للتقارير الرسمية والكشوفات
  • 🌍 دعم لغتين عربي/إنجليزي كامل (i18n)
  • 📝 Audit Log لكل تغيير حساس في البيانات
  • 🔐 أمان متقدم — JWT + RBAC + Tenant Isolation + Rate Limiting

⛔ ما لا يشمله (خارج النطاق)

  • ❌ تطبيقات جوال Native (iOS/Android) — الموقع Responsive
  • ❌ بوابة شركة مستقلة (Company Portal) — فقط تقارير ترسل عبر الإيميل
  • ❌ دفع إلكتروني أو بوابات دفع — الرواتب تصدَّر ككشف
  • ❌ ذكاء اصطناعي للتقييمات — مؤجَّل للـ Roadmap
  • ❌ محادثة مباشرة (Chat) بين المتدربين — خارج النطاق
  • ❌ نظام اختبارات إلكتروني (LMS) — مرحلة ثانية
  • ❌ تكامل مع أنظمة حكومية خارجية — يعتمد على توفر الـ API
📌
البنود المؤجَّلة مُصمَّم لها النظام من اليوم الأول بحيث يمكن إضافتها لاحقاً دون إعادة بناء.

استراتيجية التنفيذ

ابدأ من الأسفل إلى الأعلى — هرمياً

طريقة العمل الهرمية المعتمدة: ابدأ بصفحة تسجيل الدخول، ثم إدارة الموظفين والصلاحيات، ثم الشركات، ثم البرامج، ثم المتدربون — لا تبدأ بالواجهات الجميلة قبل المنطق.

الخطوة 1 — صفحة تسجيل الدخول + Auth

JWT + Refresh Token + Password Hashing. بدون login صحيح، كل ما بعده غير آمن.

الخطوة 2 — إدارة الموظفين والصلاحيات (RBAC)

إنشاء Super Admin يدوياً، ثم واجهة إضافة موظف + تعيين صلاحيات دقيقة (صفحة + إجراء + نطاق جنس + نطاق شركات).

الخطوة 3 — إدارة الشركات والأقسام

CRUD كامل. كل الحقول المطلوبة (السجل التجاري، HR، العقد، حد متدربين). تعطيل بسبب واضح.

الخطوة 4 — البرامج والمجموعات والقاعات

برنامج ينتمي لقسم، مجموعة تنتمي لبرنامج وتُسنَد لمدرب وقاعة، نوع الدوام (يومي/أيام محددة).

الخطوة 5 — التسجيل والقبول

رابط مُتحكَّم بحد أقصى، نموذج ديناميكي، مراجعة، قبول، تعيين، إرسال بيانات الدخول.

الخطوة 6 — ملف المتدرب + بوابته

ملف كامل، تحديث بيانات، تجديد هوية، رفع وثائق، بوابة شخصية تعرض البيانات الأساسية.

الخطوة 7 — الحضور

جدول المجموعة، تسجيل حضور يدوي، ثم إلكتروني، ثم ربط البصمة، ثم مطابقة، ثم استثناءات.

الخطوة 8 — الإجازات والاستئذان

مرضية برقم رسمي، سنوية بعد سنة، عامة من الإدارة، موظفين بالراتب/بدون.

الخطوة 9 — المخالفات والرواتب

خمس درجات، توقيع إلكتروني، خصم تلقائي، راتب فردي + تعديلات دائمة/شهرية.

الخطوة 10 — الإشعارات والتقارير والتميز

Cron Jobs + Templates + Queue + تصدير شامل + منصة المتميزين.

⚠️
لا تتخطَّ القاعدة الذهبية: أولوية التنفيذ هي Data Model → APIs → Workflow → UI → Reports. أي مطوِّر يبدأ بالواجهة قبل الـ API سيعيد العمل مرتين.
قابلية التوسع

مصمَّم اليوم — جاهز للغد

النظام مبني بحيث يمكن إضافة هذه الميزات مستقبلاً دون هدم البنية.

🤖 مساعد AI للإدارة

"كم متدرب غائب اليوم؟" / "اقتراح أفضل مجموعة لمتدرب جديد". مُخطَّط له في Phase 10 Roadmap.

📱 تطبيق جوال Native

الـ API جاهز للاستهلاك من أي client. تطبيق React Native يمكن بناؤه في 4 أسابيع لاحقاً.

🏢 بوابة الشركة

اليوم الشركة تتلقى تقارير عبر الإيميل. غداً يمكن إضافة بوابة حيث تراجع HR التقارير مباشرة.

💳 بوابات الدفع

الراتب يصدَّر كشف. مستقبلاً يمكن ربطه بـ STC Pay / بنك مباشرة لدفع تلقائي.

📚 نظام اختبارات (LMS)

تقييمات نهاية البرنامج، شهادات، مسارات. قابل للإضافة كـ Module مستقل.

🌐 White-Label

ألوان وشعار وroot domain لكل معهد. قاعدة البيانات مُعدَّة من اليوم الأول.
