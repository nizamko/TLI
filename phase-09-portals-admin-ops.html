ما ستُسلِّمه هذه المرحلة

المرحلة التاسعة تُحوِّل النظام الإداري الداخلي إلى منظومة مفتوحة للمستخدمين النهائيين. ثلاث بوابات متمايزة الصلاحيات — المتدرب يرى فقط بياناته، المدرب يدير مجموعاته فقط، والـ Super Admin يرى كل شيء ويتحكم في كل شيء. كل بوابة Mobile-First مع حماية صارمة على مستوى الاستعلام.

الطبقة الأمامية التي تُكمِل الدورة

المراحل الثماني السابقة بنت نظاماً إدارياً داخلياً ممتازاً — لكن بدون بوابة المتدرب، يبقى المتدرب مُتلقِّياً سلبياً يعتمد على الموظفين لكل معلومة. بوابة المتدرب تُحرِّره: هو يتابع حضوره، يطلب إجازة، يرفع وثيقة، يعترض على مخالفة — كل ذلك بنفسه من هاتفه.

📱
بدون البوابة
الوضع الحالي
  • ✗ المتدرب يتصل بالمعهد لمعرفة حضوره
  • ✗ طلبات الإجازة ورقية أو واتساب شخصي
  • ✗ الموظفون يُضيِّعون وقتهم في الاستفسارات
  • ✗ لا رؤية للـ Super Admin على صحة النظام
🚀
مع البوابة
الوضع المستهدف
  • ✓ المتدرب يرى بياناته لحظياً من هاتفه
  • ✓ طلبات الإجازة رقمية ومتتبَّعة
  • ✓ الموظفون يركّزون على الإدارة لا الاستفسارات
  • ✓ Super Admin يرى Audit Log وصحة النظام في الوقت الفعلي
🔒
مبدأ الفصل التام: كل بوابة مُصمَّمة بصلاحيات معزولة على مستوى الاستعلام — لا يُمكن لمتدرب رؤية بيانات متدرب آخر حتى لو عبث بالـ URL أو الـ token. Row-Level Security مُطبَّق في كل endpoint.

الصفحات والشاشات

21 شاشة — متدرب (10) + مدرب (5) + أدمن (6)

# الشاشة الوصف التفصيلي الدور Priority
T-01 Dashboard المتدرب نظرة سريعة: الدرس القادم، نسبة الحضور %، رصيد الإجازات، آخر الإشعارات، أي تحذيرات نشطة متدرب P0
T-02 سجل الحضور جدول شهري تفاعلي (حاضر / غائب / إجازة / متأخر) مع فلتر بالشهر، تصدير PDF متدرب P0
T-03 طلبات الإجازة نموذج طلب جديد (نوع + تاريخ + مرفق)، جدول الطلبات السابقة مع الحالة (معلق/مقبول/مرفوض) متدرب P0
T-04 الإجازة المرضية نموذج مخصص للمرضية: رقم التقرير الطبي الرسمي + رفع المستند + التواريخ متدرب P0
T-05 الملف الشخصي عرض البيانات الكاملة، تعديل البيانات غير الحساسة مباشرة، إرسال طلب تعديل للحقول الحساسة (اسم، هوية) متدرب P1
T-06 رفع الوثائق قائمة الوثائق المطلوبة (هوية، طبي، إلخ) مع مؤشر الاكتمال، رفع متعدد، معاينة، حذف متدرب P1
T-07 مركز الإشعارات قائمة مرتبة زمنياً، فلتر بالنوع (تحذير/تذكير/معلومة)، تحديد الكل كمقروء متدرب P1
T-08 المخالفات عرض القائمة (القرار + التاريخ + الدرجة)، قراءة فقط، زر "اعتراض" يفتح نموذج الاستئناف متدرب P1
T-09 التميز والإنجازات بطاقات الإنجازات، عدد مرات الترشح، الفترة الزمنية، رسالة المدرب متدرب P2
T-10 الشهادات متاحة بعد التخرج فقط — شهادة الاجتياز كـ PDF، QR code للتحقق، تاريخ الإصدار متدرب P2
TR-01 مجموعاتي بطاقات المجموعات المُعيَّنة على المدرب — اسم المجموعة، عدد المتدربين، الجدول الزمني مدرب P0
TR-02 كشف المتدربين قائمة متدربي مجموعة محددة — الاسم، نسبة الحضور، الحالة، زر اتصال مدرب P0
TR-03 تسجيل الحضور واجهة تسجيل إلكتروني — التحقق من الموقع GPS، كشف اسمي تفاعلي، تأكيد دفعي مدرب P0
TR-04 إجازاتي طلب إجازة خاص بالمدرب، سجل الطلبات السابقة مدرب P1
TR-05 ترشيحات التميز نموذج ترشيح متدرب للتميز — اختيار المتدرب، الفئة، السبب، الفترة مدرب P1
A-01 Super Admin Dashboard إجماليات النظام: عدد المستخدمين النشطين، المتدربين، أخطاء API 24h، حجم DB، Queue backlog Super Admin P0
A-02 Audit Log Viewer جدول قابل للفرز — فلاتر (مستخدم، تاريخ، كيان، إجراء)، pagination، تصدير CSV/Excel Super Admin P0
A-03 System Health Panel مؤشرات حية: DB size, Redis memory, Queue sizes, Active connections, API errors, Slow queries Super Admin P0
A-04 Backup / Restore قائمة النسخ (تلقائي/يدوي)، حالة التنفيذ، حجم الملف، زر نسخ يدوية، تنزيل، استعادة Super Admin P1
A-05 إعدادات النظام جدول Key-Value: اسم المعهد، الشعار، التواصل، القيم الافتراضية — تعديل مباشر مع تسجيل Audit Super Admin P1
A-06 Feature Flags تفعيل/تعطيل ميزات تدريجياً — نسبة الـ rollout، وصف الميزة، آخر تعديل Super Admin P2

الجداول الجديدة في المرحلة التاسعة

6 جداول جديدة تدعم التشغيل والمراقبة والضبط الديناميكي للنظام — جميعها بعناوين dh-green.

⚙️ system_settings
PK id · serial
key · varchar(100) UNIQUE NOT NULL
value · JSONB NOT NULL
category · varchar(50) — (general/notifications/portal/security)
description · text
FK updated_by → users.id
updated_at · timestamptz DEFAULT now()
💾 backups
PK id · uuid DEFAULT gen_random_uuid()
type · enum('auto','manual')
file_url · text
size_bytes · bigint
started_at · timestamptz
completed_at · timestamptz
status · enum('pending','running','success','failed')
FK triggered_by → users.id (nullable)
📊 system_health_snapshots
PK id · serial
snapshot_at · timestamptz DEFAULT now()
db_size · bigint (bytes)
redis_memory · bigint (bytes)
queue_size · int
active_connections · int
api_errors_count_24h · int
slow_queries_count · int
📋 portal_activity_logs
PK id · uuid DEFAULT gen_random_uuid()
FK user_id → users.id
portal · enum('trainee','trainer','admin')
action · varchar(100) — (view/submit/update/download)
path · varchar(255)
ip_address · inet
user_agent · text
duration_ms · int
created_at · timestamptz DEFAULT now()
🚩 feature_flags
PK id · serial
key · varchar(100) UNIQUE NOT NULL
is_enabled · boolean DEFAULT false
rollout_percentage · int DEFAULT 0 (0–100)
description · text
FK updated_by → users.id
updated_at · timestamptz DEFAULT now()
🔍 saved_searches
PK id · uuid DEFAULT gen_random_uuid()
FK user_id → users.id
name · varchar(100) NOT NULL
entity_type · varchar(50) — (trainee/violation/company/…)
filters · JSONB NOT NULL
created_at · timestamptz DEFAULT now()

العلاقات بين الجداول

# جداول المرحلة التاسعة وعلاقاتها system_settings N:1 users (updated_by — من عدَّل الإعداد) backups N:1 users (triggered_by — nullable للنسخ التلقائية) portal_activity_logs N:1 users (user_id — من نفَّذ الإجراء) feature_flags N:1 users (updated_by — آخر من غيَّر الـ flag) saved_searches N:1 users (user_id — البحث المحفوظ لمستخدم بعينه) system_health_snapshots standalone (لا FK — لقطات دورية مستقلة) # إندكسات مُوصى بها CREATE INDEX ON portal_activity_logs (user_id, created_at DESC); CREATE INDEX ON portal_activity_logs (portal, action); CREATE INDEX ON system_health_snapshots (snapshot_at DESC); CREATE UNIQUE INDEX ON system_settings (key); CREATE UNIQUE INDEX ON feature_flags (key); # GIN index للبحث الشامل CREATE EXTENSION IF NOT EXISTS pg_trgm; CREATE INDEX ON trainees USING GIN (full_name gin_trgm_ops); CREATE INDEX ON companies USING GIN (name gin_trgm_ops);
API Endpoints

28 نقطة اتصال — موزَّعة على ثلاث بوابات

🎓 بوابة المتدرب — Trainee Portal
GET /api/portal/trainee/dashboard الملخص الرئيسي: الدرس القادم، نسبة الحضور، رصيد الإجازات، آخر 3 إشعارات
GET /api/portal/trainee/attendance سجل الحضور مع ?month=&year= — يعيد مصفوفة أيام مع الحالة
GET /api/portal/trainee/leaves قائمة طلبات الإجازة مع الحالة والتواريخ
POST /api/portal/trainee/leaves تقديم طلب إجازة جديد — body: {type, from, to, reason, attachment_url?}
GET /api/portal/trainee/violations قائمة المخالفات المسجَّلة على المتدرب — read only
POST /api/portal/trainee/violations/:id/appeal تقديم اعتراض على مخالفة محددة — body: {reason, evidence_url?}
GET /api/portal/trainee/documents قائمة الوثائق المرفوعة مع الحالة (pending/approved/rejected)
POST /api/portal/trainee/documents رفع وثيقة جديدة — multipart/form-data: {type, file}
GET /api/portal/trainee/notifications قائمة الإشعارات مع فلتر ?is_read= — تحديث is_read عبر PATCH
GET /api/portal/trainee/certificates الشهادات المتاحة — يعيد 403 إذا لم يتخرج المتدرب بعد
GET /api/portal/trainee/excellence إنجازات وترشيحات التميز الخاصة بالمتدرب
GET /api/portal/trainee/profile بيانات الملف الشخصي الكاملة
POST /api/portal/trainee/profile/update-request طلب تعديل حقول حساسة — ينشئ سجل في update_requests (Phase 04)
👨‍🏫 بوابة المدرب — Trainer Portal
GET /api/portal/trainer/groups المجموعات المُعيَّن عليها المدرب الحالي فقط
GET /api/portal/trainer/group/:id/roster كشف متدربي مجموعة محددة — مع التحقق أن المدرب مُعيَّن عليها
POST /api/portal/trainer/attendance/record تسجيل حضور إلكتروني — body: {group_id, date, records[], gps_coords}
GET /api/portal/trainer/leaves إجازات المدرب + نموذج طلب جديد (POST /api/portal/trainer/leaves)
POST /api/portal/trainer/nominations ترشيح متدرب للتميز — body: {trainee_id, category, reason, period}
⚙️ عمليات Super Admin
GET /api/admin/audit-logs سجل المراجعة مع فلاتر: ?user_id=&from=&to=&entity=&action= + pagination
GET /api/admin/audit-logs/export تصدير Audit Log — نفس الفلاتر + format=csv|xlsx
GET /api/admin/system-health اللقطة الفورية لصحة النظام + آخر 24 ساعة من system_health_snapshots
GET /api/admin/system-settings جميع إعدادات النظام مُجمَّعة بالـ category
PUT /api/admin/system-settings/:key تحديث قيمة إعداد معين — يُسجِّل في Audit Log تلقائياً ويُبطل الـ cache
GET /api/admin/backups قائمة النسخ الاحتياطية مرتبة بـ started_at DESC
POST /api/admin/backups/trigger بدء نسخة احتياطية يدوية فورية — يُضاف إلى BullMQ queue
POST /api/admin/users/:id/disable تعطيل مستخدم — يُعيد 403 إذا كان Super Admin آخر
POST /api/admin/users/:id/reset-password إعادة تعيين كلمة مرور — يُرسل بريداً إلكترونياً وينهي الجلسات الحالية
GET /api/admin/search بحث شامل — ?q=&entities=trainee,company,violation — GIN + pg_trgm
GET /api/admin/feature-flags قائمة جميع Feature Flags مع حالتها الراهنة
PUT /api/admin/feature-flags/:key تحديث is_enabled + rollout_percentage لـ flag معيَّن

قواعد تشغيلية دقيقة — 11 قاعدة

🔒
قاعدة 9.1 — عزل بيانات المتدرب: جميع queries في trainee portal تتضمن WHERE trainee_id = req.user.trainee_id إلزامياً — لا توجد طريقة لتجاوزه حتى لو تم تعديل الـ request يدوياً.
👨‍🏫
قاعدة 9.2 — عزل مجموعات المدرب: SELECT * FROM groups WHERE trainer_id = req.user.trainer_id — المدرب لا يرى أي مجموعة غير مُعيَّن عليها حتى لو عرف الـ group_id.
📍
قاعدة 9.3 — شروط تسجيل الحضور الإلكتروني: GPS ضروري + يجب أن تكون الإحداثيات في نطاق 200 متر من المعهد + التسجيل مقبول فقط خلال آخر 60 دقيقة من بداية الحصة. تجاوز هذه الشروط يُعيد 422 مع رسالة توضيحية.
✏️
قاعدة 9.4 — طلبات التعديل الحساسة: تعديل الاسم الكامل أو رقم الهوية الوطنية من بوابة المتدرب لا يُطبَّق مباشرة — ينشئ سجلاً في update_requests (المرحلة الرابعة) يحتاج موافقة مشرف.
⚠️
قاعدة 9.5 — المخالفات للقراءة فقط: بوابة المتدرب تُظهر المخالفات بصيغة عرض بحتة. الزر الوحيد المتاح هو "اعتراض" الذي يُنشئ violation_appeal في المرحلة السابعة — لا يمكن حذف أو تعديل المخالفة.
🔍
قاعدة 9.6 — أداء البحث الشامل: Global Search يعمل عبر pg_trgm و GIN indexes على جداول trainees, companies, violations, documents. عتبة التشابه 0.3. الاستعلام محدود بـ 20 نتيجة لكل entity_type. Cache Redis بمفتاح search:{hash(q+entities)} TTL 60 ثانية.
📋
قاعدة 9.7 — Audit Log: pagination إلزامي (page_size max 100). تصدير CSV/Excel يُنفَّذ كـ BullMQ job لتفادي timeout. الـ export link صالح 10 دقائق فقط (signed URL).
⚙️
قاعدة 9.8 — تغيير الإعدادات: كل PUT على system_settings يُسجِّل في Audit Log (entity='system_settings', action='update', old_value, new_value) ثم يُبطل مفتاح Redis settings:{key} فوراً.
💾
قاعدة 9.9 — سياسة النسخ الاحتياطية: Cron Job يومي الساعة 02:00 بالتوقيت المحلي. الاحتفاظ بآخر 30 نسخة تلقائية (الأقدم يُحذف تلقائياً). النسخ اليدوية تُحتفظ حتى يحذفها الـ Super Admin يدوياً. Storage: S3-compatible (Backblaze B2 أو AWS).
🚩
قاعدة 9.10 — Feature Flags: الـ rollout يعمل بـ hash المستخدم: hash(user_id) % 100 < rollout_percentage — يضمن ثبات التجربة لنفس المستخدم بغض النظر عن وقت الطلب. إلغاء الـ flag (is_enabled=false) يُوقف الميزة لكل المستخدمين فوراً.
// Feature Flag evaluation — helper
function isFeatureEnabled(flagKey, userId) {
  const flag = await redis.get(`flag:${flagKey}`);
  if (!flag || !flag.is_enabled) return false;
  if (flag.rollout_percentage === 100) return true;
  const hash = crc32(userId.toString()) % 100;
  return hash < flag.rollout_percentage;
}

// GPS validation — trainer attendance
function isWithinInstitute(lat, lng) {
  const d = haversine(INSTITUTE_LAT, INSTITUTE_LNG, lat, lng);
  return d <= 200; // meters
}

// Attendance window check
const sessionStart = getSessionStart(groupId, date);
const windowEnd = addMinutes(sessionStart, 60);
if (!isWithin(now, [sessionStart, windowEnd])) {
  throw new AppError('خارج نافذة الحضور (60 دقيقة)', 422);
}

قاعدة 9.11 — Responsive Design إلزامي: جميع شاشات البوابات تستخدم Mobile-First breakpoints. بوابة المتدرب ومدرب محسَّنتان لـ 375px. الجداول تتحول إلى بطاقات في الجوال. touch targets ≥ 44px. لا horizontal scroll على أي شاشة.

المخاطر والافتراضات

ما يمكن أن يتأخر أو يُعيق

⚠️ المخاطر
  • ⚠️ أداء الجوال على 3G: صور الوثائق المرفوعة قد تثقِّل الاتصال — يتطلب lazy loading + compression.
  • ⚠️ ارتباك الأدوار: مستخدم بدورين (مدرب + مشرف) قد يرى بيانات غير متوقعة — يتطلب تعريف واضح للـ "active role" في الجلسة.
  • ⚠️ أداء البحث الشامل: عند وصول DB إلى 100K+ متدرب، pg_trgm قد يتأخر — يتطلب مراقبة مستمرة وإضافة Elasticsearch لاحقاً.
  • ⚠️ تكاليف تخزين النسخ: 30 نسخة يومية × حجم DB = تكاليف متصاعدة — يتطلب سياسة compression + تيّرد (hot/cold storage).
  • ⚠️ GPS spoofing: المدرب قد يزوِّر موقعه — يتطلب Cross-check مع IP + توثيق device fingerprint.
📌 الافتراضات
  • ✓ المتدرب يملك هاتفاً ذكياً بمتصفح حديث.
  • ✓ المعهد لديه نطاق جغرافي ثابت ومحدَّد (lat/lng + radius).
  • ✓ بنية S3-compatible متاحة للنسخ الاحتياطية.
  • ✓ Audit Log لن يُحذف أبداً (retention للأبد أو حسب السياسة التنظيمية).
  • ✓ بيانات المراحل 01-08 سليمة ومختبرة قبل البدء في البوابات.
  • ✓ الشهادات تُولَّد فقط بعد وضع الحالة = graduated في trainee profile.
